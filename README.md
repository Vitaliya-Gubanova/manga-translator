# Manga Translator — детекция пузырей/текста (YOLOv8) + OCR + перевод + редактирование

Проект автоматически находит на страницах манги речевые пузыри и текст (YOLOv8), распознаёт текст внутри пузырей (OCR), переводит его на русский язык и позволяет отредактировать перевод перед сохранением готовой картинки.

## Содержание репозитория

```
manga-translator/
├── README.md                    # этот файл
├── weights/
│   └── best.pt                  # лучшие веса обученной модели YOLOv8n
├── training/
│   ├── Yolo_V.ipynb             # код подготовки датасета и обучения YOLO
│   ├── args.yaml                # полный конфиг обучения (гиперпараметры)
│   └── metrics/                 # логи и графики обучения (см. ниже)
├── demo/
│   ├── Manga_otrisovka.ipynb    # код инференса: OCR → перевод → редактор → сохранение
│   └── demo_images/             # демонстрационные картинки манги (China / Korea)
└── data/
    └── README.md                # ссылка на размеченный датасет (Google Drive)
```

## 1. Датасет и разметка

Датасет собирался и размечался вручную с помощью программы **LabelImg**:
- 765 изображений манги в формате `jpeg` (папка `converted_images`);
- разметка на 2 класса: `0 — bubbles` (пузырь), `1 — text` (текст);
- YOLO-разметка (`.txt`) — папка `labels`.

Обе папки упакованы в архив `Labeling.zip`. Из-за размера (~126 МБ) архив не хранится в этом репозитории — ссылка на скачивание находится в [`data/README.md`](data/README.md).

## 2. Обучение модели YOLO

Код обучения: [`training/Yolo_V.ipynb`](training/Yolo_V.ipynb).

Пайплайн:
1. Монтирование Google Drive, распаковка `Labeling.zip`.
2. Объединение картинок и разметки в единую структуру, split train/val (80/20).
3. Генерация `data.yaml` (2 класса: `bubbles`, `text`).
4. Обучение `YOLOv8n` (`yolov8n.pt`) на GPU.

Ключевые параметры обучения (полный конфиг — [`training/args.yaml`](training/args.yaml)):

| Параметр | Значение |
|---|---|
| Модель | yolov8n.pt |
| Эпохи | 80 |
| Batch | 16 |
| Image size | 640 |
| Optimizer | auto |
| Время обучения | ~30 минут на GPU |

| Класс | Precision | mAP@0.5 |
|---|---|---|
| bubbles | — | 0.991 |
| text | — | 0.948 |

- Recall (все классы) = **0.98** при confidence ≈ 0.000

Графики и confusion matrix — в [`training/metrics/`](training/metrics/):
- `results.png` — кривые лоссов и метрик по эпохам (box/cls/dfl loss, precision, recall, mAP50, mAP50-95);
- `BoxPR_curve.png`, `BoxF1_curve.png`, `BoxP_curve.png`, `BoxR_curve.png` — PR/F1/Precision/Recall кривые;
- `confusion_matrix.png`, `confusion_matrix_normalized.png` — матрицы ошибок;
- `labels.jpg` — распределение разметки по классам, координатам и размерам боксов;
- `results.csv` — метрики по всем 80 эпохам в табличном виде.

По итогам обучения сохраняются:
- лучшая модель — `best.pt` (используется в этом репозитории, см. `weights/best.pt`);
- последняя эпоха — `last.pt` (в репозиторий не включена).

## 3. Демонстрационный инференс: OCR → перевод → редактирование

Код: [`demo/Manga_otrisovka.ipynb`](demo/Manga_otrisovka.ipynb).

Для запуска нужны:
- обученная модель `best.pt` (см. `weights/`);
- архив демонстрационных картинок — в этом репозитории уже распакован в [`demo/demo_images/`](demo/demo_images/): папка `China` (5 картинок манги на китайском, горизонтальный текст) и `Korea` (6 картинок манги на корейском).

### Логика работы

1. **Выбор языка** — при запуске нужно ввести `0` (Chinese Simplified) или `1` (Korean).
2. **Детекция** — обученная модель YOLO проходит по всем картинкам выбранного языка и находит пузыри и текстовые области.
3. **OCR** — текст внутри пузырей распознаётся через `easyocr` (v1.7.2). Для повышения качества применяется несколько вариантов предобработки изображения и постобработки текста (отдельно для китайского и корейского). Результат сохраняется в `ocr_result.json` (координаты пузырей, текстов и распознанный текст по каждой странице).
4. **Перевод** — распознанный текст переводится на русский через `GoogleTranslator` (`deep_translator`). Результат добавляется в `translated_result.json`.
5. **Редактирование** — интерактивный цикл:
   - вводится номер картинки, с которой начать редактирование;
   - показываются оригинал и перевод рядом, с отрисованными боксами и ID текстовых полей;
   - доступные действия: отредактировать текст по ID, сохранить как есть, завершить цикл;
   - при сохранении итоговая картинка (без боксов/ID, только чистый перевод) сохраняется в `jpeg` в `/content/translated_img_{lang_name}`.

### Используемые библиотеки

`ultralytics` (YOLOv8) · `easyocr` · `deep-translator` (Google Translate) · `opencv-python` · `Pillow` · `ipywidgets` (интерактивный редактор в Colab)

## Как воспроизвести

1. Скачать датасет по ссылке из `data/README.md`, положить в `/content/drive/MyDrive/Labeling.zip` (Google Colab).
2. Запустить `training/Yolo_V.ipynb` — обучится модель, веса появятся в `/content/runs/detect/yolo_runs/bubbles_text/weights/best.pt`.
3. Для демонстрации инференса — запустить `demo/Manga_otrisovka.ipynb`, указав путь к `best.pt` и к папке с демо-картинками (`demo/demo_images`).

> Повторное обучение не обязательно — в репозитории уже лежат готовые веса (`weights/best.pt`) и все метрики/графики обучения для ознакомления.
